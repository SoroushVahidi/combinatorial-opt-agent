# Repository Cleanup Plan

This document summarizes the cleanup of stale generated artifacts. Only **SAFE_TO_DELETE** items are removed by the script. **POSSIBLY_DELETE** items are listed for later manual review.

## Summary of disk-heavy areas (before cleanup)

| Location | Approx size | Notes |
|----------|-------------|--------|
| `results/` | 616 MB | Includes `mention_slot_scorer` (~609 MB), `paper/` (~4.8 MB), and retrieval/baseline CSVs/JSONs |
| `results/mention_slot_scorer/` | 609 MB | Training output: checkpoints + final. Only `final/` and `metrics.json` are needed at inference. |
| `data/models/retrieval_finetuned/` | 348 MB | Retrieval model: `checkpoint-10038` (261 MB) + `final/` (88 MB). Code uses `final/` only. |
| `data/` | 401 MB | Processed/raw data, external, models — **keep** (benchmark and pipeline data). |
| `data_external/` | (large) | Recently collected public datasets — **keep**. |

## What will be deleted (SAFE_TO_DELETE)

- **Intermediate training checkpoints** (not needed for current pipeline; “final” is used):
  - `results/mention_slot_scorer/checkpoint-2000` (~261 MB)
  - `results/mention_slot_scorer/checkpoint-2115` (~261 MB)
  - `data/models/retrieval_finetuned/checkpoint-10038` (~261 MB)
- **Rationale:** `retrieval/search.py` and docs reference `retrieval_finetuned/final` and `mention_slot_scorer/final` only. Intermediate checkpoints are regenerable by re-running training.

**Total estimated space freed:** ~783 MB.

## What will be kept

- **Source code:** All `.py`, configs, scripts.
- **Docs:** All `docs/*.md`, manuscript-related files.
- **Current pipeline outputs:**
  - `results/paper/*` — downstream tables, plots, LaTeX, summary CSV (used by docs and `make_nlp4lp_paper_artifacts.py`).
  - `results/*.csv`, `results/*.json` — retrieval metrics, baselines (used by paper artifacts and reproducibility).
  - `results/mention_slot_scorer/final/`, `results/mention_slot_scorer/metrics.json`.
  - `data/models/retrieval_finetuned/final/`.
- **Data:** `data/raw`, `data/processed`, `data/catalogs`, `data/sources`, `data/external`, `data_external/`.
- **Manifests and logs:** `data_external/manifests/`, collection logs.

## What will NOT be deleted (POSSIBLY_DELETE — manual review later)

- **`logs/`** — Slurm job logs (`mention_slot_%j.out`, `mention_slot_%j.err`). Small (~257 KB). Optional to remove if no longer needed.
- **`results/harder_eval_latest.zip`** — Archive from older “harder eval” experiment. Remove if that experiment is abandoned.
- **`__pycache__/`** under `collectors/`, `tests/`, `retrieval/`, `tools/` — Generated bytecode; safe to delete for clutter, but omitted from automated cleanup to avoid touching many paths. Can run `find . -type d -name __pycache__ -exec rm -rf {} +` (excluding venv/data_external) if desired.

## Rationale

- **Current methods** we care about: typed, constrained, semantic_ir_repair, optimization_role_repair, acceptance_rerank, hierarchical_acceptance_rerank. Their outputs live in `results/paper/` and in the downstream summary CSV; none of these are removed.
- **Checkpoints:** Only “final” model dirs are referenced by code; intermediate checkpoints are for resumable training and can be re-created.
- No benchmark splits, metric definitions, or source files are changed or removed.

## Manifest of deleted paths

See **`docs/archive/REPO_CLEANUP_DELETED_FILES_MANIFEST.txt`** for the exact list of paths passed to the cleanup script (one path per line).

## Cleanup script

Run from repo root:

```bash
bash scripts/cleanup_old_artifacts.sh
```

The script only deletes paths listed in the manifest, prints each path before deletion, and skips nonexistent paths.
