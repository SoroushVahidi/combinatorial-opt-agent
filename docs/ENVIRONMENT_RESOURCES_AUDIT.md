## 9. Risks / missing access / uncertain items

- **HF:** If udell-lab/NLP4LP is gated or private, missing or invalid token will cause _load_hf_gold to fail. No token is in repo; user must set env.
- **GAMSPy:** data_private is gitignored; GAMS examples and license are local. New clone or another machine may lack them; license must be activated per doc.
- **Empty evals:** nl4opt_family_eval_test.jsonl, resocratic_eval.jsonl (and possibly nl4opt_comp_eval.jsonl) are documented as empty; scripts that expect them may skip or fail.
- **Paper tables:** make_nlp4lp_paper_artifacts.py uses a fixed baseline list; acceptance_rerank and optimization_role_repair are not in the final table CSV (see RESULTS_VS_CODE_VERIFICATION.md).
- **Shell limits:** Past audits noted “fork: Resource temporarily unavailable” on login nodes; heavy runs should use compute nodes (e.g. sbatch).
